What this app does

mediacleanuparr connects to Radarr (movies) and Sonarr (TV) and helps you reclaim disk space by removing low-rated, oversized, and empty/orphaned media — safely. It never deletes anything on its own:

  • Dry-run-only is on by default. Scans only ever propose — nothing is touched until you turn dry-run-only off in Setup.
  • Every real deletion makes you type DELETE and confirm.
  • It only ever acts inside your mounted media folder, never a system path or a media root itself.

The tabs, explained

Setup

Where you connect everything and set the safety switches. Enter your Radarr, Sonarr and TMDb (required for rating scans) URLs and API keys and test each one. Optionally connect Overseerr/Jellyseerr ("Seerr"). The deletion-behaviour toggles here control what actually happens when you confirm a deletion (see How blocking works below).

Scan & review

The rating-based cleanup. Pick a minimum user rating (0–100, the TMDb audience score ×10); anything scoring below it is flagged. Choose scope (Movies / TV / Both), optionally Include unrated and Clean up empty (0-byte) items & folders, then Run dry scan. Review the list, tick/untick rows, type DELETE and confirm. Titles with no rating or a rating of 0 are marked needs review and are never auto-selected. Each row also has an Exclude box.

Biggest items

Find your largest titles by disk size regardless of rating — great for clearing big shows you never watch. Pick scope and how many to show (up to 500). Turn on Remove empty items & orphaned folders to also surface every 0-byte entry (no files on disk) and orphaned empty folders; those are pre-selected because there's nothing on disk to lose. Tick what to purge, type DELETE, confirm.

Logs & reports

A full audit trail of every action it has taken, plus downloadable JSON/CSV reports of each deletion run (saved under /config/reports).

How re-download blocking works

When you remove something, the app can also stop Radarr/Sonarr (and import lists) from silently grabbing it again. What it does depends on the type and your Setup toggles. The Re-download blocked column on a scan shows whether blocking will be applied to each row.

  • Movies (Radarr) — with "Radarr: add import exclusion" on (default), the movie is added to Radarr's Import List Exclusions when removed, so Radarr and your import lists won't re-add or re-grab it.
  • TV (Sonarr), with files — the episode files are deleted and, with "Sonarr: unmonitor flagged series" on (default), the series is unmonitored (series + all seasons) but kept in Sonarr, so it stops auto-downloading.
  • Empty / 0-byte entries — the stale entry is removed from Radarr/Sonarr (no files to delete) and the same import exclusion is applied, blocking auto re-download.
  • Orphaned folders — just an empty directory deleted from disk. No Radarr/Sonarr entry and no blocking involved.
  • Overseerr / Jellyseerr (optional) — if connected, the matching request is cleared so Seerr stops auto-requeuing it. Important: the title stays fully re-requestable — it is not blacklisted.

Separately, the per-row Exclude checkbox is not a block — it only tells this app to skip that title on future scans. It does nothing to Radarr/Sonarr.

Unblocking & putting a title back

Nothing here is permanent. To restore something:

  • A blocked movie (Radarr) — in Radarr go to Settings → Lists → Import List Exclusions (or the movie editor), remove its exclusion, then add the movie back and search.
  • An unmonitored series (Sonarr) — open the series in Sonarr and turn monitoring back on for the series and the seasons you want, then trigger a search. (Nothing was excluded; it was just unmonitored.)
  • An empty series that was removed (Sonarr) — if it was removed with an exclusion, clear it under Settings → Import Lists → Import List Exclusions, then re-add the series.
  • A cleared Seerr request — just request the title again in Overseerr/Jellyseerr; it was never blacklisted.
  • A title you excluded from scans — open the Scan & review tab and remove it from the "Excluded from scans" list at the bottom.

Rule of thumb: re-download blocking lives in Radarr/Sonarr (exclusion or unmonitor), so undoing it is done there. Scan exclusions live in this app and are removed here.

Automatic unblock on re-request (Seerr)

Optional, and the easiest way to undo a block. With Auto-unblock when re-requested in Seerr turned on in Setup and the webhook configured, you don't have to touch Radarr/Sonarr by hand — when a human re-requests a previously-removed title in Overseerr/Jellyseerr, this app lifts its own block automatically.

On a matching re-request it will:

  • Remove the Radarr import exclusion (movies) or Sonarr import-list exclusion (removed series).
  • Re-monitor the title if it's still in Radarr/Sonarr.
  • Re-add it and trigger a search if it had been removed entirely — so the request is fulfilled even if the exclusion briefly blocked Seerr's own add.

On a re-request it clears the matching Radarr/Sonarr exclusion — whether this app set it or it predates the app — then re-monitors and re-adds & searches. (So an exclusion you added by hand is also cleared if that title is re-requested.) To set up: enable the toggle in Setup, click Save, then paste the generated Webhook URL into Overseerr/Jellyseerr → Settings → Notifications → Webhook (trigger on request events). The token in the URL keeps it private.
